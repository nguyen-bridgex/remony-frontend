# Remony 見守りサービス管理システム

## プロジェクト概要

このプロジェクトは、高齢者や要介護者の見守りサービスを管理するためのWebアプリケーションです。Next.js、Firebase、Tailwind CSSを使用して構築されており、利用者の登録、管理、アラート設定などの機能を提供します。

## 主な機能

### 🏠 ホームダッシュボード
- システム全体の概要を表示
- 各機能へのクイックアクセス

### 👤 利用者登録・管理
- 新しい利用者の登録
- 個人情報（名前、住所、連絡先など）の管理
- バイタル機器の設定
- 通知先の設定（メール・LINE）

### 📊 利用者一覧
- 登録済み利用者の一覧表示
- 検索機能
- 利用者情報の詳細表示

### 🏥 事業所管理
- 事業所の新規登録・編集・削除
- 事業所情報の一覧表示
- 詳細設定項目：
  - 基本情報（事業所名、住所）
  - API連携設定（サービスアカウント、クライアントID、クライアントシークレット）
  - チャットボット設定（ボット番号、トークンURL、API Base URL）
  - セキュリティ設定（プライベートキー）
- 利用者と事業所の紐付け管理

### ⚙️ アラート設定
- 心拍数アラートの設定
- 表体温アラートの設定
- 歩数アラートの設定
- 距離アラートの設定
- 睡眠アラートの設定
- カロリー消費アラートの設定
- 発電量アラートの設定

### 🎨 UI/UXの特徴
- **カスタマイズ可能な背景テーマ**: 利用者詳細ページでは6つの異なる色テーマから選択可能
- **拡張された検索機能**: 利用者一覧ページでより広い検索バーでユーザビリティを向上
- **カスタム404エラーページ**: 分かりやすいエラーメッセージと自動リダイレクト機能
- **レスポンシブデザイン**: すべてのデバイスで最適化された表示

## セットアップ手順

### 前提条件
- Node.js (v16以上推奨)
- npm または yarn

### 1. リポジトリのクローン
```bash
git clone [リポジトリURL]
cd remony-frontend
```

### 2. 依存関係のインストール
```bash
npm install
```

### 3. 環境変数の設定
プロジェクトルートに `.env.local` ファイルを作成し、以下の環境変数を設定してください：

```env
NEXT_PUBLIC_LAMBDA_API_URL=your_api_endpoint_url
```

### 4. 開発サーバーの起動
```bash
npm run dev
```

開発サーバーが起動すると、ブラウザで `http://localhost:3000` にアクセスできます。

## 利用可能なスクリプト

```bash
# 開発サーバーの起動
npm run dev

# プロダクションビルド
npm run build

# プロダクションサーバーの起動
```

## 主要なページ

### ホームページ (`/`)
- システムのメインダッシュボード
- 各機能へのナビゲーション

### 利用者登録 (`/user/register`)
- 新しい利用者の登録フォーム
- 個人情報、バイタル機器設定の入力

### 利用者一覧 (`/users`)
- 登録済み利用者の一覧表示
- 検索・フィルタリング機能

### 事業所管理 (`/hospitals`)
- 事業所の登録・編集・削除機能
- 事業所一覧の表示（カード形式）
- 包括的な事業所情報管理：
  - **基本情報**: 事業所名（必須）、住所
  - **API連携設定**: サービスアカウント、クライアントID、クライアントシークレット
  - **チャットボット設定**: ボット番号、トークンURL、API Base URL
  - **セキュリティ設定**: プライベートキー（マルチライン対応）
- 利用者と事業所の紐付け管理
- レスポンシブモーダルデザイン（スクロール対応）
- バリデーション機能付きフォーム

### 利用者詳細 (`/user/[id]`)
- 特定の利用者の詳細情報表示
- 複数の背景色テーマから選択可能（ブルー、グリーン、パープル、ピンク、オレンジ、ティール）
- リアルタイムでテーマ変更が可能

### アラート設定 (`/user/[id]/setting`)
- 利用者のアラート設定管理
- 各種バイタルデータの閾値設定

### 404エラーページ (`/404`)
- 存在しないページへのアクセス時に表示
- 自動的にホームページへリダイレクト（10秒カウントダウン）
- 手動でホームページまたは前のページに戻る機能
- 日本語での分かりやすいエラーメッセージ

## 開発ガイドライン

### コードスタイル
- TypeScriptを使用し、型安全性を確保
- ESLintとPrettierでコード品質を維持
- コンポーネントは関数型コンポーネントを使用

### スタイリング
- Tailwind CSSを使用
- レスポンシブデザインを考慮
- アクセシビリティに配慮

### API通信
- Axiosを使用したHTTP通信
- エラーハンドリングを適切に実装
- 環境変数でAPIエンドポイントを管理
- 主要なエンドポイント：
  - ユーザー管理（登録、取得、更新、削除）
  - 事業所管理（CRUD操作、利用者紐付け）
  - アラート設定（各種バイタルデータ設定）
